3D Lunar Topography: Interactive Moon Surface Map

High-Resolution 3D Visualization using NASA LROC Data & Python

Can you spot the Tycho crater from this visualization?

Overview

This project generates an interactive 3D topographic map of the Moon using high-resolution Digital Elevation Model (DEM) data from NASA's Lunar Reconnaissance Orbiter (LRO). By processing the massive WAC_GLD100 dataset (approx. 118m/pixel resolution at the equator), this tool transforms raw planetary data into a navigable 3D globe.

The visualization leverages Python's geospatial ecosystem to handle large .cub raster files, implementing custom spherical transformation algorithms and Plotly for high-performance rendering. This allows users to explore lunar craters, maria, and highlands in 3D directly in a web browser.

Objectives

  • Objective 1: Process and downsample massive planetary DEM datasets (WAC_GLD100) for efficient visualization.
  • Objective 2: Implement spherical coordinate transformations to map 2D raster data onto a 3D globe.
  • Objective 3: Create an interactive, shader-based 3D model that highlights elevation differences using lighting effects.

Methodology

Step Description
1. Data Ingestion [cite_start]Loaded the .cub format DEM using GDAL and converted it to an Xarray DataArray for efficient handling[cite: 6, 11].
2. Downsampling & Cleaning [cite_start]Downsampled the massive array (originally 54,585 x 109,166 pixels) to a manageable size (step=70) and filtered extreme outliers.
3. Processing [cite_start]Applied Gaussian filtering (sigma=2) to smooth pixelation and performed histogram equalization for better visual contrast.
4. Spherical Transformation [cite_start]Mapped the 2D latitude/longitude grid onto a 3D sphere using trigonometric transformation ($x = R \cdot \cos(lat) \cdot \cos(lon)$, etc.).
5. Rendering [cite_start]Rendered the 3D surface using Plotly, applying lighting effects (ambient, diffuse, specular) to simulate solar illumination on craters.

# To the 3D Map (The Fix)

# STEP 1: DOWNSAMPLE FIRST 
# We select the stride (resolution) HERE to save RAM.
# step = 500 is good for a smooth interactive web model.
final_step = 250 

# Create the working DEM from the subset immediately. 
# We DO NOT convert the whole 22GB 'arr' to float32.
dem = arr[::final_step, ::final_step].astype(np.float32)

# STEP 2: CLEANING 
# Now we are only processing a few megabytes, so this is fast and safe.
dem[dem <= -25000] = np.nan
dem[dem >=  25000] = np.nan

# Calculate percentile on the already-small array
lo, hi = np.nanpercentile(dem, [.5, 99.5])
dem = np.clip(dem, lo, hi)

# Apply filter to smooth out pixelation from downsampling
dem = gaussian_filter(dem, sigma=2)
In[38]:
# STEP 3: NORMALIZATION
dem_min, dem_max = np.nanmin(dem), np.nanmax(dem)
dem_norm = (dem - dem_min) / (dem_max - dem_min)
# Equalize histogram for better visual contrast
dem_eq = exposure.equalize_hist(np.nan_to_num(dem_norm, nan=0.5))
In[39]:
# STEP 4: SPHERICAL TRANSFORMATION
ny, nx = dem.shape
lons = np.linspace(-180, 180, nx) * np.pi/180
lats = np.linspace(90, -90, ny) * np.pi/180
lon_grid, lat_grid = np.meshgrid(lons, lats)

R0 = 1.0
# Increase exaggeration slightly to make craters "pop"
exaggeration = 0.05 
R = R0 + exaggeration * (np.nan_to_num(dem_norm, nan=0.5) - 0.5)

X = R * np.cos(lat_grid) * np.cos(lon_grid)
Y = R * np.cos(lat_grid) * np.sin(lon_grid)
Z = R * np.sin(lat_grid)
In[]:
# STEP 5: PLOTLY RENDERING 
moon = go.Surface(
    x=X, y=Y, z=Z,
    surfacecolor=dem_eq, 
    colorscale="gray",
    cmin=0, cmax=1, 
    showscale=False,
    lighting=dict(ambient=0.5, diffuse=1, specular=0.25, roughness=0.9),
    lightposition=dict(x=2, y=2, z=1.5)
)

halo = go.Surface(
    x=1.02*X, y=1.02*Y, z=1.02*Z,
    surfacecolor=np.ones_like(dem_eq),
    colorscale=[[0, "rgba(255,255,255,0.10)"], [1, "rgba(255,255,255,0)"]],
    showscale=False, opacity=0.15
)

fig = go.Figure([moon, halo])

fig.update_layout(
    scene=dict(
        xaxis=dict(visible=False), 
        yaxis=dict(visible=False), 
        zaxis=dict(visible=False),
        aspectmode="data", # 'data' usually looks better for spheres than 'cube'
        bgcolor="black"
    ),
    margin=dict(l=0, r=0, t=0, b=0),
    paper_bgcolor="black", 
    plot_bgcolor="black",
    autosize=True
)

filename = f"moon_dem_shaded_{final_step}.html"
fig.write_html(filename, include_plotlyjs="cdn", full_html=True, config={"responsive": True})

print(f"Success! 3D Map saved to: {filename}")
fig.show()

Results & Insights

This tool successfully renders a lightweight, interactive 3D globe from gigabytes of raw data.

  • Crater Depth Visualization: The vertical exaggeration factor ($0.05$) successfully makes impact craters "pop" for better visibility[cite: 119].
  • Elevation Extremes: The analysis identified the lowest point at -8,184m and the highest peak at 10,522m (values approximate after downsampling) [cite: 60-61].
  • Interactive Exploration: The final HTML output allows for rotation, zooming, and panning across the lunar surface[cite: 153].
